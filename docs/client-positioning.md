# Client Positioning

## What You Are Selling

Do not sell this as “advanced structural analysis”.

Sell it as proof that you can build:
- interactive engineering calculators
- browser-based explainers
- technical training visualizers
- custom pre-design or concept-stage tools
- engineering software with a modern web UX

## The Pitch

WebJenga shows that you can take engineering logic written in C++ and turn it into a fast browser experience with:
- real-time interaction
- visual explanation
- clean UI
- deployable web delivery

That is valuable even when the model is intentionally simplified.

## Best Client Types

- structural engineering consultancies
- geotechnical consultants
- product manufacturers with technical sales tools
- training providers
- universities or engineering educators
- firms with internal spreadsheet tools that need a better UI

## Problems You Can Solve

- replace opaque spreadsheets with interactive browser tools
- build technical visualizers for proposals or client communication
- create internal training tools for junior engineers
- build product explainers for manufacturers
- make concept-stage calculators easier to understand and safer to use

## Messaging Rules

- be explicit about model scope
- never imply full FEM if it is not FEM
- emphasize clarity, speed, and communication value
- show the app, then explain the custom tools you could build from the same stack

## Short Positioning Statement

I build browser-based engineering tools that turn technical calculations into interactive visual experiences. WebJenga is a simple example: C++ engineering logic compiled to WebAssembly, visualized live in the browser, with a UX designed for explanation rather than opaque spreadsheet use.

## Outreach Angle

Useful angles:
- “If you have a spreadsheet-based engineering tool that clients or junior staff struggle to interpret, I can turn it into an interactive browser app.”
- “If you need a technical explainer or concept-stage calculator for sales, training, or pre-design work, I can build it.”
- “If you want to keep your engineering logic fast and robust, I can use C++/WebAssembly where that makes sense.”
